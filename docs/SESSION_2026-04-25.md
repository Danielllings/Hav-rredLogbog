# Session 25. april 2026 - Vejr-migration & UI-opdateringer

## Hovedændring: DMI → Open-Meteo migration

### Problem
DMI's EDR API (opendataapi.dmi.dk) var ekstremt langsom:
- **3-16 sekunder** per forecast-kald (CoverageJSON format)
- Rate limiting (429 errors) ved parallelle requests
- Forkerte parameternavne (`wind-speed` i stedet for `wind-speed-10m`) gav tomme svar
- Krævede Cloud Functions proxy med cold start latency

### Løsning
Migreret al vejrdata til **Open-Meteo** (open-meteo.com):
- **~200-250ms** responstid (60x hurtigere)
- Bruger DMI's eget HARMONIE-model via `models=dmi_seamless` (2km opløsning)
- Direkte fra telefon til API - ingen proxy, ingen API key
- Gratis, 10.000 kald/dag

### Hvad blev migreret

| Komponent | Før (DMI) | Nu (Open-Meteo) |
|-----------|-----------|------------------|
| **Spot-vejr** (`dmi.ts`) | 3 parallelle DMI EDR kald, 3-16s | 2 parallelle Open-Meteo kald, ~250ms |
| **Tur-evaluering** (`dmi.ts`) | DMI Climate + Ocean stations (nærmeste station, kan være 20km væk) | Open-Meteo med `start_date`/`end_date` på turens GPS-position (2km grid) |
| **Havstrøm-overlay** (`CurrentVelocityOverlay.tsx`) | DMI proxy → CoverageJSON, 15s timeout | `fetchCurrentGrid()` via Open-Meteo, ~400ms |
| **Bølge-overlay** (`WaveSwellOverlay.tsx`) | DMI proxy → CoverageJSON, 15s timeout | `fetchWaveGrid()` via Open-Meteo, ~400ms |
| **Vind-overlay** (`WindOverlay.tsx`) | DMI proxy → 2 parallelle WAM kald, 8s timeout | `fetchWindGrid()` via Open-Meteo, ~300ms |
| **Vandstand-overlay** (`WaterLevelOverlay.tsx`) | DMI proxy → CoverageJSON + GeoJSON fallback | `fetchWaterLevelGrid()` via Open-Meteo, ~400ms |
| **Salinitet-overlay** (`SalinityHeatmapOverlay.tsx`) | DMI proxy | **DMI direkte** (Open-Meteo har ikke salinitet) |

### Nye filer
- `lib/openMeteoGrid.ts` - Fælles grid-helper til overlay-komponenter. Genererer 18×18 punkt-grid, kalder Open-Meteo med komma-separerede koordinater, returnerer CoverageJSON-kompatibelt format.

### Vejrparametre (spot-vejr)

**Weather API** (`api.open-meteo.com/v1/forecast`):
- Temperatur, vindstyrke, vindretning, vindstød
- Luftfugtighed, tryk (hPa), skydække, nedbør
- Dugpunkt, sigtbarhed

**Marine API** (`marine-api.open-meteo.com/v1/marine`):
- Bølgehøjde, bølgeperiode, bølgeretning
- Vandtemperatur (SST), vandstand, havstrøm

### Tur-evaluering
- Bruger `start_date`/`end_date` for turens præcise tidsvindue
- Ture < 90 dage: forecast API med `models=dmi_seamless`
- Ture > 90 dage: archive API (`archive-api.open-meteo.com`)
- `computeCoastWindInfo()` uændret (ren GPS-matematik)

### DMI stadig brugt til
- **Salinitet** (eneste overlay der stadig bruger DMI EDR direkte)
- **Stationsdata** (`OCEAN_STATIONS_DK`) til kort-pins i spot-weather

## UI-ændringer

### Tracking-skærm (`BentoTrackingDashboard.tsx`)
- **"Fangst!"** → **"Registrer fangst"** (DA) / **"Register catch"** (EN)
- Tre stablede knapper i stedet for 2-i-1 sekundær række:
  - **Registrer fangst**: Fuld bredde, grøn pille (`#15803D`)
  - **Mål vandtemp**: Fuld bredde, blå pille (`#1D4ED8`)
  - **Stop**: Fuld bredde, rød muted
- Haptic feedback: `notificationAsync(Success)` ved fangst-registrering
- Kort bruger native satellit (`mapType="satellite"`) i stedet for orto-tiles

### Vejrkort (`spot-weather.tsx`)
- **Long press** for at placere markør (forhindrer utilsigtede markeringer ved panorering)
- **Haptic feedback** (`impactAsync(Medium)`) når markør placeres
- Standard kort = native Apple/Google Maps, satellit = native satellit
- Fjernet DMI stations-markører fra kort
- Fjernet Mapbox/orto-tiles fra hele appen

### Galleri / Fangstdetaljer
- **Personlig rekord-bar**: Strakt til fuld bredde (fjernet `marginHorizontal: 16`)
- **Ikoner opdateret** (konsistent på tværs af alle sider):
  - Længde: `ruler` (MaterialCommunityIcons) i stedet for `resize`
  - Vægt: `weight-kilogram` (MaterialCommunityIcons) i stedet for `scale`/`barbell`

### Statistik (`statistics.tsx`)
- **Spot-præstation**: Beholder best/worst kort + tilføjet top 2-5 som kompakte piller nedenunder
- Sorteret efter fisk/tur rate (et spot med 1 fangst på 1 tur overgår et spot med 0 fangster på 10 ture)

### Kort-migration (hele appen)
Fjernet Mapbox/dataforsyningen orto-tiles fra alle skærme:
- `index.tsx` (tracking)
- `BentoTrackingDashboard.tsx` (mini-kort under tracking)
- `trip-replay/[id].tsx` (tur-replay)
- `catch/[id].tsx` (fangstdetaljer)

Alle bruger nu native `mapType="satellite"` i stedet.
